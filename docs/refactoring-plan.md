## Принципы

- Сначала стабилизируем поведение и тестовый контур, потом меняем архитектуру.
- Не смешиваем крупный рефакторинг с изменением бизнес-логики без явной необходимости.
- Каждая итерация должна оставлять проект в рабочем состоянии.
- Python ETL и R-пайплайн nowcast/fizob считаются одной продуктовой цепочкой данных.

## Приоритеты

- `P0`: защита от потери данных, регрессий и неработающего CI.
- `P1`: уменьшение связности и дублирования.
- `P2`: улучшение структуры проекта, документации и developer experience.

- Повторяющаяся логика TNVED и EDIZM удалена из отдельных процессоров.
